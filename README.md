# Weather App

## Overview
The Weather App is a web application built to practice working with APIs and handling API keys securely. The project focuses on fetching real-time weather data and displaying it in a clean, user-friendly interface.

This project was also an opportunity to improve frontend styling efficiency and UI design skills.

---

## Purpose of the Project
The main goals of this project were:

- To learn how to fetch data from external APIs
- To understand how API keys work and how to use them securely
- To practice handling JSON responses
- To improve JavaScript skills (DOM manipulation & event handling)
- To design a clean and efficient UI

---

## 🛠️ Technologies Used

- HTML5 – Structure
- CSS3 – Styling
- JavaScript (ES6) – Logic and API integration
- Weather API – Real-time weather data
- Figma – UI/UX design and layout planning

---

## Design

The interface was designed using Figma before development.  
The goal was to create a minimal, modern, and responsive layout that clearly displays:

- Temperature
- Humidity
- Weather conditions
- "Feels like" temperature
- Additional weather details

---

## Features

- Search weather by city
-  Displays Temperature 
-  Displays humidity percentage
-  Responsive design

---

## How It Works

1. User enters a city name
2. The app sends a request to the weather API
3. The API returns weather data in JSON format
4. The app extracts and displays relevant information dynamically

---

##  What I Learned

- How to use `fetch()` to retrieve API data
- How to handle asynchronous JavaScript
- How to manipulate and display API data dynamically
- How to structure a clean UI using Figma
- The importance of separating logic and presentation

---

## API Key Usage

The application requires a weather API key to function.  
Make sure to keep your API key secure and avoid exposing it publicly in production environments.
